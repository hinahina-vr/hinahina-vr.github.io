# ワークスペースルール

## 日記作成

> [!CAUTION]
> 以下のキーワードが会話に含まれたら、**何よりも先にワークフローを読むこと**。手動でデータを書いてはいけない。
> トリガーワード: `日記`, `元ネタ`, `diary`, `daily-context`, `今日のメモ`

日記を書く・日記の作成を依頼された場合は、必ず以下の2つのワークフローを参照し、その手順に従うこと：
1. `.agents/workflows/diary.md` — 作成フロー（データ収集〜プッシュ）
2. `.agents/workflows/diary-rules.md` — キャラ別ガイドライン・フォーマット

**手順の要点:**
1. まず `npm run auth:daily-sources` でChromeを起動し、ユーザーにXログインしてもらう
2. `collect-daily-context.mjs` でX・Swarm・Healthデータを自動収集する
3. 自動収集されたデータを元に日記を作成する（手動でSwarm/Xデータを捏造しない）

## 勉強コーナー作成

> [!CAUTION]
> 以下のキーワードが会話に含まれたら、**先に勉強コーナーのルールを読むこと**。
> トリガーワード: `勉強`, `study`, `力学`, `章`, `老中AI`, `大奥AI`, `study-drafts`

勉強コーナーの記事作成・更新を依頼された場合は、必ず以下を参照すること：
1. `.agents/workflows/study-rules.md` — 勉強コーナー鬼才ルール

**手順の要点:**
1. まず `study-drafts/{subject}/{nn}_下書き.md` を作るか更新する
2. 打ち合わせMDで方向性、担当AI、図解案を固める
3. `study/{subject}/{nn}.md` を更新し、`npm run build:study` でHTMLを生成する
4. `npm run check:study` とブラウザテストを通す

## 再発防止策（2026-03-18制定）

> [!CAUTION]
> 以下のチェックを**すべて通過するまで**日記の完成を報告してはならない。

### 1. ファイル名管理
- 本編日記 `diary/YYYY-MM-DD_下書き.md` は、本文確定後に `diary/YYYY-MM-DD_タイトル.md` にリネームすること
- キャラ日記の `daily-context` 内 `### 元ネタ` のパスも正式ファイル名に合わせて更新すること

### 2. 文字数チェック（必須）
- 各キャラ日記の本文（見出し `# ...` 行〜 `---` 区切り直前まで）が **1000文字以上** であることを確認する
- 確認方法: PowerShellで全ファイルの文字数を計測し、1000文字未満のファイルが0件であることを確認してから完了報告する
- **1000文字未満の日記を放置して次に進んではならない**

### 3. 話題割り当てチェック
- 計画フェーズで決定した「キャラ × 話題」の割り当て表を `task.md` に記録する
- 各キャラ日記を書くとき、割り当てられた話題を本文の主軸にすること
- `daily-context` の `### 本日の話題` に、割り当てられた話題IDとタイトルを記載すること
