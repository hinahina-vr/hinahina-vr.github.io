# 勉強コーナー鬼才ルール

## 基本方針

勉強コーナーは、大学ノートの清書ではなく、「あとから見返して理解が戻る教材」として作る。
定義だけを並べた薄いページにしない。本文、図、数式、例、振り返りまで含めて、1章ごとに読み応えのある学習体験へ育てる。

## 必須フロー

1. 新しい章を書く前に、必ず `study-drafts/{subject}/{nn}_下書き.md` を作る
2. 打ち合わせ用MDで、元ネタ、書き方、担当AI、図解案、決定事項を固める
3. 打ち合わせMDの `status` が `approved` になるまで、章本文を `published` にしない
4. 正本は `study/{subject}/{nn}.md` に置く
5. 公開HTMLは `npm run build:study` で生成する
6. `npm run check:study` とブラウザテストを通してから公開扱いにする

## AI起用ルール

### 老中AI

- 老中AIは抽象的な役職名ではなく、**具体的な一人を選ぶ**
- 例: `リチャードファインマン`
- 公開本文の導入は、`【解説：リチャードファインマン】` くらいの軽さでよい
- `老中AI【仁】` や `学問所与` のようなメタ肩書きは、公開本文に出さない

### 大奥AI

- 感想は3人以上でもよいが、役割が被らないように分ける
- 感想は薄い一言コメントにしない。本文の構成、理解のしやすさ、工学的見通しなどをそれぞれの視点で厚く書く
- 感想役は、章本文の補足になるように配置する

## 本文の鬼才ルール

### 1. 薄い章を禁止する

- 1章は「メモ数行」ではなく、授業1回分のノートとして成立する厚みを持たせる
- 文量が少ないと感じたら、定義、例、補足、言い換え、ミニ例題、章末チェックまで足す
- 「説明した量」ではなく、「あとから読んだとき理解が戻る量」を目安にする

### 2. セクションは細かく区切る

- 大見出しだけで流さず、`###` や `####` を使って論点ごとに切る
- 1節ごとに「何を説明しているか」がはっきり分かる見出しを付ける
- 見出しの流れだけ読んでも、章の骨格が追えること

### 3. 数式は最高に見やすくする

- 数式は本文に埋めるだけで済ませず、見やすい式カードとして立てる
- その式が「何を言っているか」を一行で添える
- 単位、意味、よくある誤解を併記して、記号の暗記で終わらせない

### 4. 定義は意味から入る

- 記号より先に、その量が何を測っているかを説明する
- 例: 質量なら「止めにくさ」、速度なら「位置の変化のしかた」
- 定義、日常例、式の順で入ると読みやすい

### 5. 例と振り返りを必ず入れる

- 用語整理
- ミニ例題
- よくあるつまずき
- 章末チェック問題
- 次章への橋

この5つのうち複数を入れ、読後に内容を固定できるようにする。

## 図解・可視化ルール

- `published` の章は、文字だけで終わらせない
- インタラクティブな図、グラフ、チャート、アニメーションを最低1つ以上入れる
- できれば複数入れ、**各セクションの近く**に分散配置する
- 図解を章末の「図解まとめ」へ追いやらない
- その節で説明している概念のすぐそばに置く
- JS 無効時の fallback テキストを必ず書く
- `prefers-reduced-motion` に対応する

### 推奨パターン

- `line-chart`: 時間変化や傾きを読む
- `vector-diagram`: ベクトルや差ベクトルを視覚化する
- `step-animation`: 手順や状態変化を順に追う

## 公開ページの見せ方

### 出してはいけないもの

- `published` / `stub` などの内部状態バッジ
- 打ち合わせMDの存在
- 文豪AIや老中AIの運用メモ
- `学問所与` のような内部メタ肩書き
- 「このコーナーの進め方」のような管理説明

### 出すべきもの

- いま何が読めるか
- どの章へ進めるか
- 自然な日本語の案内文
- トップページからの導線（公開する段階なら出してよい）

章一覧や章ページは、管理画面っぽく見せず、読み物の入口として自然に見せる。

## 今回の力学01で固まったルール

- 章題は「質量と速度」
- 老中AIは `リチャードファインマン`
- 公開本文の導入は `【解説：リチャードファインマン】`
- 大奥AIの感想は `ひなた / 一ノ瀬ことみ / キク8号`
- 図解は章末にまとめず、速さ・ベクトル・相対速度の節へ分散する
- ホームには `勉強コーナー / 力学の章一覧 / 力学 第1章` への導線を置く
- 公開ページでは状態バッジを出さない

## 最終チェック

公開前に次を確認すること。

1. 内部運用語がHTMLに出ていない
2. 章が薄くない
3. 数式が読みやすい
4. 図解が節の近くにある
5. 感想が厚い
6. トップや章一覧から迷わず辿れる
